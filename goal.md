# 基于Web门户的轻量级SSL VPN访问

## 任务背景
在企业远程办公场景中，员工往往只需通过浏览器访问内部的Web应用（如OA系统、邮件系统、报表平台），无需访问底层的TCP端口。传统的全功能SSL VPN部署复杂且成本高，而本系统旨在构建一个轻量级的Web型SSL VPN网关。它利用标准的HTTPS协议，为用户提供安全的Web登录门户，并在认证成功后，通过反向代理技术安全地转发用户请求至内网Web服务器。该系统无需安装任何客户端插件，即可实现“浏览器即客户端”的安全接入，满足基本的远程办公安全需求。

## 基础素材
1. **参考标准**：
   - RFC 8446 (TLS 1.3) / RFC 5246 (TLS 1.2)
   - OWASP Top 10 (Web应用安全风险)

2. **测试环境**：
   - 模拟内网：搭建1-2个简单的 Web服务（如“内部新闻站”、“文件下载页”），仅允许网关访问，不对外网开放。
   - 模拟外网：普通笔记本电脑或手机浏览器。

3. **技术选型方向**：可采用合适的Web框架、网络请求处理库及加密相关技术，也可结合主流Web服务器软件辅助构建。

## 功能描述
1. **HTTPS安全门户**：
   - 系统强制启用HTTPS，配置SSL证书，确保登录页面及后续通信的加密。
   - 提供简洁的用户登录界面，支持身份认证。

2. **应用级反向代理**：
   - 用户登录成功后，门户首页展示其有权访问的内网应用列表。
   - 点击链接后，系统在后台建立与服务器的连接，将用户的浏览器请求转发至内网目标URL并将响应结果返回给用户。
   - 关键特性：用户浏览器地址栏始终显示网关地址，隐藏内网真实的IP和端口。

3. **细粒度访问控制**：
   - 基于角色资源的可见：不同用户登录后，看到的内网应用列表不同。
   - 会话超时的控制：用户无操作超过指定时间自动退出登录。

4. **安全审计日志**：
   - 记录用户登录时间、源IP、访问的内网应用URL及操作结果。提供管理员视图，可查看近期的访问日志。

## 核心方法
- **加密传输**：利用公钥基础设施保障浏览器到网关链路的机密性。
- **应用层反向代理**：在应用层截获并转发流量，而非网络层，实现无需客户端的便捷访问。
- **基于角色的访问控制**：将用户身份与内网资源URL进行绑定映射。

## 实现路径
1. **架构设计**：
   - 采用B/S架构。服务端作为唯一的入口，监听相应端口。
   - 核心流程：用户请求 → HTTPS网关 → 身份验证 → 权限检查 → 发起内部HTTP请求 → 返回结果给用户。

2. **核心功能规划**：
   - 涵盖HTTPS配置、认证与会话管理、反向代理逻辑、权限控制等模块。

3. **前端界面**：
   - 开发简单的登录页和仪表盘，仪表盘动态渲染用户有权限的应用图标/链接。

## 测试用例设计
1. **加密通道测试**：验证客户端与网关之间的流量是否为加密流，确保无法直接看到明文信息。
2. **权限隔离测试**：验证无权限用户尝试访问受限资源时，系统是否能进行有效拦截并记录。
3. **会话安全测试**：验证会话的有效性管理机制，以及超时后自动登出功能是否正常。
