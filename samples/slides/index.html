🗄 数据库模型 — Sea-ORM 2.0

5 张核心表，sqlite::memory: 测试隔离，14 个单元测试

  • users — id/username/password_hash/role/created_at
  • sessions — UUID v4 令牌，8h 过期自动清理
  • apps/user_app_permissions — RBAC 应用级权限映射
  • audit_logs — 登录/代理/拒绝操作记录

📡 后端框架 — Pingora ProxyHttp

Cloudflare 开源的异步代理框架，实现核心反向代理逻辑

  • early_request_filter — 拦截 API 路径、静态文件、HTML 页面
  • upstream_peer — 身份认证 → RBAC 检查 → 动态选择内网目标
  • fail_to_proxy — 优雅处理 API/Page 路径（非真实错误）

🔒 认证 — argon2 + UUID Session

handle_login 完整流程：验证密码 → 创建会话 → 审计日志 → Set-Cookie

  • argon2.verify_encoded() 密码哈希验证
  • Uuid::new_v4() 生成 128bit 随机会话令牌
  • HttpOnly + SameSite=Lax + Max-Age Cookie
  • create_audit_log 记录登录事件（成功/失败分开记录）

📊 eBPF 流量监控 — TC Classifier

tc_ingress: 统计接收字节 + TCP 连接追踪 (SYN/FIN/RST)

  • ctx.load 从网络包头读取 ether_type / tot_len / proto / flags
  • BYTES_RECV HashMap 累加 IP 包长度
  • CONN_COUNT Array: SYN 递增, FIN/RST 递减

📈 StatusCollector — 实时状态

请求计数 / 字节累加 / 会话追踪, get_stats() 返回 SystemStats

  • record_request 累加 bytes_sent/recv（不再丢弃参数）

📊 eBPF 加载器 — try/fallback 模式

优雅降级：BPF 二进制加载失败时自动回退到 StatusCollector

  • init_ebpf() 读取 bpf-linker 编译的 ELF → TC attach
  • 失败时 EbpfMonitor { ebpf: None } → read_stats 返回零值
  • /api/status 中 eBPF 有数据时优先，否则用应用层计数

🔐 RBAC 访问控制

  • user — user_has_app_permission 检查授权
  • 无权限 → 403 + audit_log 记录
